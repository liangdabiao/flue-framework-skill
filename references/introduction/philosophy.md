Flue is designed for agents that do work rather than only answer questions. A model becomes useful in an application when it is given a deliberate harness for context, tools, sessions, and safe execution.

## Agent = model + harness

A harness is the programmable TypeScript boundary where your application decides which capabilities exist and how they are controlled.

## Write once, deploy anywhere

The same authored agents and workflows can target local Node development or deployed platform environments while preserving their core application model.
